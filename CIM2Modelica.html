cim2modelica cim.map

This transformation algorithm maps CIM v16 data into OpenIPSL Modelica library components for power systems. The proposed solution establishes initial guesses of the variables in the Modelica power system components for static and dynamic analysis.

The main algorithm follows a similar procedure as described in An efficient method of extracting network information from CIM asset model by Shukla S, Yao mark G. The source CIM profile information contains the topology and the power flow results, in conjunction with the dynamic information of the components. In summary it contains the following profiles:

  • Equipment
  • Topology
  • State-Variable
  • Dynamic

The transformation algorithm builds the network component with mapping. For example:

Terminal - ConductingEquipment => ConductingEquipment

Terminal - TopologicalNode => Bus

EnergyConsumer => Load

PowerTransformerEnd - Terminal => Transformer
